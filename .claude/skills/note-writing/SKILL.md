# Note Writing Skill

## 概要
noteプラットフォーム向けの記事作成を支援するスキルです。読者に価値を提供する質の高いコンテンツ作成をサポートします。

## 対象プラットフォーム
- **サービス名**: note (https://note.com)
- **フォーマット**: Markdown（note形式）
- **出力先**: `note-articles/` ディレクトリ

## 主な機能

### 1. 記事構成の設計
- ターゲット読者の明確化
- 記事の目的・ゴール設定
- 構成・アウトラインの作成
- 見出し階層の最適化

### 2. コンテンツ作成
- 導入部（フック）の作成
- 本文の論理展開
- 具体例・エピソードの挿入
- まとめ・CTA（行動喚起）の設計

### 3. SEO・発見性の最適化
- キーワード選定
- タイトルの最適化
- ハッシュタグの提案
- メタディスクリプションの作成

### 4. トーン・スタイル調整
- カジュアル / ビジネス / 技術解説など
- 読者層に合わせた文体
- 専門用語の平易化
- 読みやすさの調整

## 利用可能コマンド

| コマンド | 説明 |
|---------|------|
| `/user:note-write` | 記事の全文を作成 |
| `/user:note-outline` | 記事のアウトラインを生成 |
| `/user:note-seo` | SEO最適化の提案を生成 |
| `/user:note-refine` | 既存記事のブラッシュアップ |

## 記事テンプレート

### ハウツー記事
```markdown
# [課題解決]を実現する[方法/手順]

## はじめに
- 読者の悩み・課題の明確化
- この記事で得られる価値

## [ステップ1]: [見出し]
- 具体的な手順
- スクリーンショット・図解
- ポイント・注意点

## [ステップ2]: [見出し]
...

## まとめ
- 振り返り
- 次のアクション提案

#ハッシュタグ #タグ
```

### 体験談・ストーリー記事
```markdown
# [体験・結果]から学んだ[学び]

## きっかけ
- 背景・状況説明
- なぜ始めたのか

## 実践したこと
- 具体的な行動
- 苦労したこと
- 工夫したこと

## 得られた結果・学び
- 変化・成果
- 気づき・学び
- 読者へのメッセージ

#ハッシュタグ #タグ
```

### 技術解説記事
```markdown
# [技術・概念]を[対象読者]向けに解説

## この記事の対象読者
- 前提知識
- 読了後に得られるスキル

## [技術・概念]とは
- 定義
- なぜ重要なのか

## 実装・使い方
```コードブロック
// サンプルコード
```
- 解説
- ポイント

## まとめ
- 要点の整理
- さらなる学習リソース

#プログラミング #技術
```

## 記事作成の基本原則

### 1. 読者ファースト
- 読者の悩み・疑問に答える
- 具体的で実践的な内容
- 専門用語は必ず説明

### 2. 構成の明確さ
- 見出しで内容が予測できる
- 1段落1メッセージ
- 適切な改行・余白

### 3. 信頼性の確保
- 事実と意見の区別
- 情報源の明記（必要に応じて）
- 自身の経験・実績の提示

### 4. 視覚的な工夫
- 見出しの階層化
- 箇条書きの活用
- 引用・強調の適切な使用
- 画像・図解の挿入（テキスト内で説明）

## noteの特徴に合わせた工夫

### ハッシュタグ戦略
- 3〜5個程度が適切
- 一般的なタグと固有タグの組み合わせ
- トレンドタグの活用

### 文字数の目安
- ショート記事: 500〜1,500文字
- 標準記事: 1,500〜3,000文字
- ロング記事: 3,000文字以上

### 有料記事の設計
- 無料部分で価値を示す
- 有料部分の内容を明示
- 適切な価格設定（100〜500円が一般的）

## 出力ファイル命名規則

```
note-articles/
  └── YYYY-MM-DD-記事タイトル-slug.md
```

例: `2026-01-04-TypeScript型安全ガイド-typescript-type-safety.md`

## 記事メタデータ

各記事ファイルの先頭にYAML frontmatterを含める：

```yaml
---
title: "記事タイトル"
date: 2026-01-04
category: "技術" # 技術/ビジネス/ライフスタイル/エッセイなど
tags:
  - TypeScript
  - プログラミング
  - 初心者向け
target_audience: "TypeScriptを学び始めた開発者"
estimated_reading_time: 5 # 分
is_paid: false # 有料記事の場合true
---
```

## SEOチェックリスト

- [ ] タイトルに検索キーワードを含む（28〜60文字）
- [ ] 冒頭100文字で記事の価値を伝える
- [ ] 見出しに検索意図に合ったキーワードを含む
- [ ] 内部リンク・外部リンクを適切に配置
- [ ] 関連ハッシュタグを付与
- [ ] 画像にalt属性相当の説明を記載

## 記事公開前チェックリスト

- [ ] 誤字脱字のチェック
- [ ] 事実関係の確認
- [ ] リンク切れのチェック
- [ ] スマホ表示での読みやすさ確認
- [ ] タイトルの魅力度確認
- [ ] アイキャッチ画像の準備
- [ ] 公開日時の設定

## note記事の種類別ガイド

### テキスト記事
- 最も一般的な形式
- Markdownで記述
- 画像・動画の埋め込み可能

### つぶやき
- 短文投稿（Twitter的）
- 500文字以内
- 気軽な情報発信

### 音声・動画
- 音声・動画ファイルのアップロード
- テキスト説明を併記
- 文字起こしの追加で発見性向上

### マガジン
- 複数記事をまとめる
- テーマ別に整理
- 定期購読の設定可能

## アナリティクス活用

記事公開後の改善ポイント：

- 閲覧数の推移
- スキ・コメントの傾向
- 流入経路の分析
- 人気記事の共通点抽出
